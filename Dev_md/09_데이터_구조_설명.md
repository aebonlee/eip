# 데이터 구조 설명

모든 정적 데이터는 `src/data/` 폴더에 JavaScript 파일로 관리됩니다.

---

## 파일 구조

```
src/data/
├── written-exam-data.js          # 필기시험 메인 (certTypes + import 통합 + 유틸함수)
├── chapters/                     # 필기 이론 챕터 (과목별 분리)
│   ├── engineer.js               # 기사 5과목 25챕터
│   ├── industrial.js             # 산업기사 4과목 16챕터
│   └── functional.js             # 기능사 3과목 9챕터
├── mock-questions/               # CBT 모의시험 문제 (자격증별 분리)
│   ├── engineer.js               # 기사 80문제
│   ├── industrial.js             # 산업기사 60문제
│   └── functional.js             # 기능사 45문제
├── practical-exam-data.js        # 실기시험 (SQL 15 + 알고리즘 15 + 단답형 30)
├── coding-exercises.js           # 코딩실습 (C 12 + Java 12 + Python 12)
└── lectures-data.js              # 유튜브 강의 목록
```

---

## 1. written-exam-data.js (필기시험 데이터)

분리된 챕터/문제 파일을 import하여 통합 export합니다.

### certTypes - 자격증 정보
```javascript
{
  engineer: {
    id: 'engineer',
    name: '정보처리기사',
    description: '국가기술자격 정보처리 분야 기사급',
    subjects: [
      { id: 'sw-design', name: '소프트웨어 설계', chapters: 5 },
      { id: 'sw-dev', name: '소프트웨어 개발', chapters: 5 },
      { id: 'db', name: '데이터베이스 구축', chapters: 5 },
      { id: 'sys-integration', name: '프로그래밍 언어 활용', chapters: 5 },
      { id: 'info-security', name: '정보시스템 구축관리', chapters: 5 },
    ],
  },
  industrial: { ... },  // 4과목 × 4챕터
  functional: { ... },  // 3과목 × 3챕터
}
```

### chapters - 챕터별 이론 콘텐츠 (50챕터)
`chapters/engineer.js`, `chapters/industrial.js`, `chapters/functional.js`에서 import하여 spread로 통합.

```javascript
// 각 챕터 파일 구조
export const engineerChapters = {
  'sw-design': [
    {
      id: 'ch1',
      name: '요구사항 확인',
      topics: ['소프트웨어 생명주기 모델', ...],
      content: '## 소프트웨어 생명주기 모델\n...',  // 마크다운
    },
    // ... 5챕터
  ],
  'sw-dev': [...],  // 5챕터
  'db': [...],      // 5챕터
  // ...
}

// written-exam-data.js에서 통합
export const chapters = {
  ...engineerChapters,
  ...industrialChapters,
  ...functionalChapters,
}
```

| 자격증 | 과목 수 | 과목당 챕터 | 총 챕터 |
|--------|---------|------------|---------|
| 정보처리기사 | 5 | 5 | 25 |
| 정보처리산업기사 | 4 | 4 | 16 |
| 프로그래밍기능사 | 3 | 3 | 9 |
| **합계** | | | **50** |

### mockQuestions - CBT 모의시험 문제 (185문제)
`mock-questions/engineer.js`, `mock-questions/industrial.js`, `mock-questions/functional.js`에서 import.

```javascript
// 각 문제 파일 구조
export const engineerQuestions = [
  {
    id: 1,
    subject: 'sw-design',
    question: '문제 텍스트',
    options: ['보기1', '보기2', '보기3', '보기4'],
    answer: 1,              // 정답 인덱스 (0부터)
    explanation: '해설 텍스트',
  },
  // ...
]

// written-exam-data.js에서 통합
export const mockQuestions = {
  engineer: engineerQuestions,     // 80문제 (과목별 16문제)
  industrial: industrialQuestions, // 60문제 (과목별 15문제)
  functional: functionalQuestions, // 45문제 (과목별 15문제)
}
```

### 유틸 함수
| 함수 | 용도 |
|------|------|
| `getQuestionsBySubject(certType, subjectId)` | 과목별 문제 필터링 |
| `getMockTestQuestions(certType, count)` | 랜덤 셔플 모의시험 문제 |

## 2. practical-exam-data.js (실기시험 데이터)

### sqlQuestions - SQL 실습 문제 (15문제)
```javascript
{
  id: 'sql-1',
  title: 'SELECT 기본',
  difficulty: 'easy',              // easy / medium / hard
  question: '문제 설명',
  hint: '힌트 텍스트',
  answer: 'SELECT name FROM ...',  // 정답 SQL
  alternativeAnswers: [...],       // 대체 정답 (대소문자 등)
  explanation: '해설',
  tableSchema: 'Student(id, name, dept)',  // 테이블 구조
}
```

주요 토픽: SELECT, WHERE, ORDER BY, DISTINCT, GROUP BY, HAVING, JOIN (INNER/LEFT), INSERT, UPDATE, DELETE, 서브쿼리, CASE WHEN, 집계함수

### algorithmQuestions - 알고리즘 문제 (15문제)
```javascript
{
  id: 'algo-1',
  title: '버블 정렬 결과',
  difficulty: 'easy',
  question: '문제 (여러 줄 가능)',
  answer: '3, 5, 1, 2, 8',
  alternativeAnswers: ['[3, 5, 1, 2, 8]'],
  hint: '힌트',
  explanation: '해설',
}
```

주요 토픽: 버블/선택/삽입/퀵 정렬, 스택/큐 연산, 이진 탐색, 후위 표기법, 해시, 트리 순회, 재귀, 행렬, 그리디, BFS/DFS

### shortAnswerQuestions - 단답형 문제 (30문제)
```javascript
{
  id: 'short-1',
  category: '소프트웨어 공학',
  question: '문제',
  answer: '스크럼',
  alternativeAnswers: ['Scrum'],
  hint: '힌트',
}
```

주요 카테고리: 소프트웨어 공학, 데이터베이스, 네트워크, 보안, 프로그래밍, 디자인 패턴, UML, 테스트, 프로세스 관리, 페이지 교체, OSI

## 3. coding-exercises.js (코딩 실습 데이터)

### languages - 지원 언어 정보
```javascript
{
  c: { id: 'c', name: 'C', monacoId: 'c', icon: 'C' },
  java: { id: 'java', name: 'Java', monacoId: 'java', icon: 'J' },
  python: { id: 'python', name: 'Python', monacoId: 'python', icon: 'P' },
}
```

> **코드 실행**: Wandbox API 사용 (gcc-head / openjdk-head / cpython-head)

### exercises - 언어별 문제 (각 12문제, 총 36문제)
```javascript
{
  c: [
    {
      id: 'c-1',
      title: '두 수의 합',
      difficulty: 'easy',         // easy / medium / hard
      description: '문제 설명',
      input: '3 5',               // 테스트 입력
      expectedOutput: '8',         // 기대 출력
      template: '...',            // 초기 코드 템플릿
      solution: '...',            // 정답 코드
      hint: '힌트',
    },
    // C: 12문제, Java: 12문제, Python: 12문제
  ],
}
```

| 언어 | easy | medium | hard | 주요 토픽 |
|------|------|--------|------|----------|
| C | 3 | 5 | 4 | 문자열, 배열 정렬, 포인터, 구조체, 재귀, 행렬, 연결리스트 |
| Java | 2 | 5 | 5 | 배열 정렬, 클래스/객체, 상속, ArrayList, HashMap, 예외처리, 인터페이스, 재귀, 회문 |
| Python | 3 | 5 | 4 | 문자열, 리스트 정렬, 딕셔너리, 클래스, 파일처리, 람다/map, 재귀, 정규표현식 |

### 유틸 함수
| 함수 | 용도 |
|------|------|
| `getExercises(language)` | 언어별 문제 목록 |
| `getExerciseById(language, id)` | 특정 문제 조회 |

## 데이터 확장 가이드

### 챕터 추가
1. 해당 자격증의 챕터 파일 (`src/data/chapters/engineer.js` 등) 열기
2. 과목 ID 키 아래 배열에 새 챕터 객체 추가
3. `content` 필드에 마크다운 형식 텍스트 작성 (`##` 대제목, `###` 소제목, `- **용어**: 설명`)
4. `certTypes`의 해당 과목 `chapters` 수도 함께 업데이트

### 모의시험 문제 추가
1. 해당 자격증의 문제 파일 (`src/data/mock-questions/engineer.js` 등) 열기
2. 배열에 같은 형식의 객체 추가
3. `id`는 순차적으로 부여 (기존 최대값 + 1)
4. `subject`는 해당 자격증의 과목 ID와 일치해야 함
5. `answer`는 0-indexed (0, 1, 2, 3)

### 실기/코딩 문제 추가
1. 해당 데이터 파일 (`practical-exam-data.js` 또는 `coding-exercises.js`) 열기
2. 배열에 같은 형식의 객체 추가
3. `id` 값은 고유하게 설정 (예: `sql-16`, `algo-16`, `c-13`)
