# 데이터 구조 설명

모든 정적 데이터는 `src/data/` 폴더에 JavaScript 파일로 관리됩니다.

---

## 1. written-exam-data.js (필기시험 데이터)

### certTypes - 자격증 정보
```javascript
{
  engineer: {
    id: 'engineer',
    name: '정보처리기사',
    description: '국가기술자격 정보처리 분야 기사급',
    subjects: [
      { id: 'sw-design', name: '소프트웨어 설계', chapters: 5 },
      // ... 5과목
    ],
  },
  industrial: { ... },  // 4과목
  functional: { ... },  // 3과목
}
```

### chapters - 챕터별 이론 콘텐츠
```javascript
{
  'sw-design': [
    {
      id: 'ch1',
      name: '요구사항 확인',
      topics: ['소프트웨어 생명주기 모델', ...],
      content: '## 소프트웨어 생명주기 모델\n...',  // 마크다운 형태
    },
    // ... 5챕터
  ],
}
```

### mockQuestions - CBT 모의시험 문제
```javascript
{
  engineer: [
    {
      id: 1,
      subject: 'sw-design',        // 과목 분류
      question: '문제 텍스트',
      options: ['보기1', '보기2', '보기3', '보기4'],
      answer: 1,                    // 정답 인덱스 (0부터)
      explanation: '해설 텍스트',
    },
    // ... 20문제
  ],
}
```

### 유틸 함수
| 함수 | 용도 |
|------|------|
| `getQuestionsBySubject(certType, subjectId)` | 과목별 문제 필터링 |
| `getMockTestQuestions(certType, count)` | 랜덤 셔플 모의시험 문제 |

## 2. practical-exam-data.js (실기시험 데이터)

### sqlQuestions - SQL 실습 문제
```javascript
{
  id: 'sql-1',
  title: 'SELECT 기본',
  difficulty: 'easy',              // easy / medium / hard
  question: '문제 설명',
  hint: '힌트 텍스트',
  answer: 'SELECT name FROM ...',  // 정답 SQL
  alternativeAnswers: [...],       // 대체 정답 (대소문자 등)
  explanation: '해설',
  tableSchema: 'Student(id, name, dept)',  // 테이블 구조
}
```

### algorithmQuestions - 알고리즘 문제
```javascript
{
  id: 'algo-1',
  title: '버블 정렬 결과',
  difficulty: 'easy',
  question: '문제 (여러 줄 가능)',
  answer: '3, 5, 1, 2, 8',
  alternativeAnswers: ['[3, 5, 1, 2, 8]'],
  hint: '힌트',
  explanation: '해설',
}
```

### shortAnswerQuestions - 단답형 문제
```javascript
{
  id: 'short-1',
  category: '소프트웨어 공학',      // 카테고리
  question: '문제',
  answer: '스크럼',
  alternativeAnswers: ['Scrum'],   // 영문 대체 답
  hint: '힌트',
}
```

## 데이터 확장 가이드

### 문제 추가 방법
1. 해당 데이터 파일 (`src/data/`) 열기
2. 배열에 같은 형식의 객체 추가
3. `id` 값은 고유하게 설정 (예: `sql-7`, `algo-6`)
4. `answer` 인덱스는 0부터 시작

### 챕터 콘텐츠 추가
- `chapters` 객체에 과목 ID를 키로 하여 챕터 배열 추가
- `content` 필드에 마크다운 형식 텍스트 작성
- `##`는 대제목, `###`는 소제목, `- **굵게**:` 는 용어 설명
