# 2026-07-18 정답 가림 자가 테스트 모드 (UI 기능 라운드 1)

## 배경
- 범위 분석에서 나온 도입 컨셉 중 1순위 채택: 이론 챕터의 "확인 문제"가 정답·해설을 바로 노출해 자가 테스트가 안 되던 문제 해결. (컨셉만 참고, 구현 전부 자체)

## 구현 (MarkdownContent.tsx + course.css + 스터디 페이지 3종)
- `MarkdownContent`에 `hideAnswers` prop 추가: "정답…"/"풀이:"로 시작하는 줄(리스트·문단)과 헤더에 "정답" 열이 있는 표를 `AnswerBlur`로 감싸 흐림 처리(blur 6px). **클릭하면 그 항목만 공개** — 문제 단위 자가 채점 가능.
- `useHideAnswers` 훅(localStorage `eip_hide_answers` 유지) + `AnswerToggleButton` 컴포넌트 export — 토글 상태가 페이지 이동 후에도 유지됨.
- 적용 페이지 3종: 필기 `ChapterStudy`(이 참에 자체 중복 렌더러 제거 → 공용 MarkdownContent로 통합), 실기 이론 `PracticalTheoryStudy`, 코딩 이론 `CodingTheoryStudy`.
- CSS(course.css): .answer-toggle-btn(pill 토글, active 시 primary), .answer-blur(블러+호버 시 살짝 완화+클릭 커서, revealed 시 해제).

## 효과
- 챕터 내 확인 문제 260+ 문항 전부가 즉시 자가 테스트 문제은행으로 전환됨 (기존 콘텐츠 수정 0).

## 검증
- tsc -b 에러 0, vite build 성공. 렌더러 통합으로 ChapterStudy 중복 코드 약 50줄 제거.
- 화면 확인은 대표 직접: 챕터 학습 화면 상단 "정답 가리기 (자가 테스트)" 버튼 → 확인 문제의 정답 줄이 흐려지고 클릭 시 개별 공개.
