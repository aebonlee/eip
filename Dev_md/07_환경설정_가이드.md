# 환경설정 가이드

## 1. 프로젝트 설치

```bash
cd D:\dreamit-web\eip
npm install
```

## 2. 환경변수 설정 (.env)

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
```

> **참고**: 코드 실행은 Wandbox API를 사용하며, API 키가 필요 없습니다.

### Supabase 설정
1. [Supabase](https://supabase.com) 프로젝트 생성
2. Project Settings > API에서 URL과 anon key 복사
3. Authentication > URL Configuration에서 Site URL 설정: `https://eip.dreamitbiz.com`
4. Authentication > URL Configuration에서 Redirect URL 추가: `https://eip.dreamitbiz.com/auth/callback`
5. Authentication > Providers에서 Google, Kakao 활성화

### Google OAuth 설정
1. [Google Cloud Console](https://console.cloud.google.com)에서 프로젝트 생성
2. OAuth 2.0 Client ID 생성
3. 승인된 리디렉션 URI: `https://<supabase-project>.supabase.co/auth/v1/callback`
4. Supabase Dashboard > Auth > Providers > Google에 Client ID/Secret 입력

### Kakao OAuth 설정
1. [Kakao Developers](https://developers.kakao.com)에서 앱 생성
2. 카카오 로그인 활성화 + Redirect URI 설정
3. Supabase Dashboard > Auth > Providers > Kakao에 REST API Key/Secret 입력

## 3. Supabase DB 테이블 생성

Supabase SQL Editor에서 실행:

```sql
CREATE TABLE sb_profiles (
  id UUID REFERENCES auth.users PRIMARY KEY,
  nickname TEXT,
  avatar_url TEXT,
  target_cert TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE sb_study_progress (
  id BIGSERIAL PRIMARY KEY,
  user_id UUID REFERENCES auth.users,
  cert_type TEXT,
  subject_id TEXT,
  chapter_id TEXT,
  completed BOOLEAN DEFAULT FALSE,
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE sb_test_results (
  id BIGSERIAL PRIMARY KEY,
  user_id UUID REFERENCES auth.users,
  cert_type TEXT,
  test_type TEXT,
  score INTEGER,
  total INTEGER,
  answers JSONB,
  taken_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE sb_code_submissions (
  id BIGSERIAL PRIMARY KEY,
  user_id UUID REFERENCES auth.users,
  language TEXT,
  exercise_id TEXT,
  code TEXT,
  result TEXT,
  is_correct BOOLEAN,
  submitted_at TIMESTAMPTZ DEFAULT NOW()
);
```

## 4. 개발 서버 실행

```bash
npm run dev
```
→ http://localhost:5173

## 5. 프로덕션 빌드

```bash
npm run build
```
→ `dist/` 폴더에 빌드 결과물 생성

## 6. OG 이미지 생성

```bash
npm run generate-og
```
→ `public/og-image.png` 생성 (1200x630)

## 7. 배포 (GitHub Pages)

```bash
npm run build
npx gh-pages -d dist
```

### 배포 설정 사항
- **GitHub Pages**: `gh-pages` 브랜치에서 서빙
- **커스텀 도메인**: `eip.dreamitbiz.com` (CNAME 설정)
- **SPA 라우팅**: `public/404.html`에서 리다이렉트 처리 (GitHub Pages는 SPA를 네이티브 지원하지 않음)
